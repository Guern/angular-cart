Angular-cart

Shopping cart in AngularJS

Welcome to Angular-Cart

This is my attempt to simplify e-commerce by using a shopping cart controlled entirely by JavaScript via AngularJS.

Currently, it loads product information from a JSON file via Angular's $http service and loads the data into an array. Angular then sends this information to the view so that the user can interact with products and add them to the shopping cart.

When the user is ready to check-out, Angular sends the data via POST and allows the developer to handle the information with their preferred method.

Currently, angular-cart is proof of concept only.

Instructions

To test angular-cart, simply download the project files into your htdocs directory (for MAMP/XAMPP/LAMPP) or any directory your local development server will recognize as localhost.

Using the default directory structure, visit

localhost/angular-cart/

You will be taken immediately to the products page.

Angular-cart loads the products from a file called includes/products.json and then generates a view for the user that includes a "BUY" button. When the user clicks the "BUY" button, they are presented an option of how many they want to order and they can add them to the cart.

Angular-cart also includes sidebar-widgets that are enabled by default. Once a user has an item in the cart, they can see their most recent item on the page sidebar.

They also have the option of using a drop-down menu at the top of the page to see their cart.

Modifying the files is simple. You can add new products to the JSON file as objects. Simply copy one of the pre-existing objects and edit the values or follow the same relative format.

Recent Commits

11/30/2014

Changed checkout.php to retrieve the $_POST variable via AJAX. The design philosophy has changed to make it easier to check-out. Instead of sending the user to another page, another modal will pop-up with the user's total and give them the option to check-out that way. I've updated index.html and app.js to reflect this. You can see the response information in the console.

While the demo uses PHP, the philosophy is to make this easy to implement with any back-end system.

Current Issues

Right now the cart data doesn't persist beyond the product page. However, the unimplemented solution to this is to use HTML5's localstorage or perhaps IndexedDB or another method to store the cart on the client side in order to persist the data at least until the session expires.

The current demo doesn't use product images, either, and this would be ideal for most e-commerce sites.
